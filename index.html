Introduction

Imagine coding a 3D printer live on stage...

  • Every pause risks melted plastic dripping
  • Every function shapes real material (no undo!)

Why Livecoding is Hard

  • Speed & material: FDM printing is time-sensitive, unpredictable
  • Cognitive load of creating complex forms on the fly procedurally
  • ...also combining forms; starting, stopping seamlessly

Introducing gridlib

A library for continuous path-based 3D printing.

  • Designed for livecoding I3DP
  • Presets, modifiers, timelines
  • works with vizlib visualiser

Core Techniques

  • builds forms using presets of 2D, point-based shapes (mostly)
  • modifies them via functions like sine, cosine, sawtooth, and triangle waves over time*

*time in points iterated, not musical or physical time

Challenges & Reflections

  • Experimental, under development
  • Crossfading dissimilar shapes is tricky
  • How to modify points without breaking the shape?
    • Can change the speed between points but changing speed and time breaks shapes
    • Continuous polygons (Vase mode spirals) vs. specific layers
    • Balance: aesthetics vs reliability

gridlib = new toolkit for live interactive 3D printing.
